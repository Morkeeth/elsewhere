# Elsewhere UI Flow: Decision Twin

## 1. Reopen a decision

```text
“What decision keeps reopening?”
```

The user starts from a question, a recent decision-log entry, or a selected complaint-book pattern.

**Key UI:** a quiet inbox of open decisions, not a chatbot.

## 2. Gather the record

```text
Reality      What is true or estimated?
Research     What have I already noticed?
Unknowns     What am I assuming?
```

The user selects a small number of relevant artifacts. Each becomes a visible source card with purpose, freshness, and remove control.

## 3. Name the real futures

Current Elsewhere accepts two to four paths. The public wedge starts with an honest A/B comparison and adds another path only when the user has a real one.

## 4. Add context lenses

```text
Financial resilience              [built in]
Belonging and relationships       [built in]
My model of Mum’s concern         [user-authored]
My future self’s freedom           [user-authored]
```

Opening a custom lens asks only:

1. What does this perspective protect?
2. What evidence supports your model of it?
3. What is unknown or likely to have changed?

The UI never asks “What would they say?”

## 5. Rehearse the world

The main screen contains:

```text
Two-to-four future stories
        ↓
Shared evidence receipt
        ↓
Disagreement matrix: values × futures
        ↓
Assumption map: fact / estimate / unknown
```

Selecting a shock visibly recomputes the timelines and recolors the matrix.

## 6. Find the load-bearing assumption

Instead of declaring a winner, Elsewhere asks:

```text
“Which assumption changes the decision if it is false?”
```

The user can mark it:

- Ask someone
- Read a document
- Try a small version
- Wait for a signal

## 7. Exit into an experiment

The experiment card remains constrained:

- fourteen days
- visible maximum cost
- physically actionable today
- reversible
- observable signals

## 8. Return with evidence

At the end of the experiment:

```text
What did I expect?
What happened?
What surprised me?
Which assumption changed?
```

This creates a calibration record and updates the decision’s research archive. The system becomes more useful because the user’s model gets better. It does not pretend to predict more.
